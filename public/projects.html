    Using SSH on a Nintendo 3DS

    Tech: 3DS, DevKitPro, C, Libssh2, Libsctru, 3ds-mbedtls, Claude AI for help

    The SSH tool is BudgieSSH: https://github.com/Maxbrick/budgieSSH. BudgieSSH must be built by the user using DevKitPro. Using DevKitPro, I cross-compiled the main.c file for BudgieSSH with libssh2 and 3ds-mbedtls, with my $SOURCE being 3dsvars.sh. I was unfamiliar with all of the above when I started, but I really wanted to use Ollama on my 3DS—which I was eventually able to accomplish, although I found it not particularly useful in practice. However, I did learn more about building .3dsx files, cross-compiling, and gained a better understanding of C and its use of headers.

    EventViewer Troubleshooting

    Tech: EventViewer, Claude AI, BIOS, Screwdriver.

    This project is really just me troubleshooting my machine after having issues with my CPU temp getting up to 90 degree C on startup, at least this is what my BIOS was showing, it would then shut down after about 3 minutes. This had started after getting my other project setup "OpenWebUI - Building Full Conversation AI ChatBot Using GPU". The issue was that I had acciently modified my BIOS settings on a restart without realizing it. The setting changed was my UEFI BIOS EZ Mode Fan Setting was set to Optimize, which was causing my fans to overclock on startup. Another issue that definintly was casuing problems was that my CPU was no longer seated corretly. This was most likely casued by my dog's tail slapping the side of my PC almost daily and I had to really looked in my PC case for a few months. So after reseating CPU and changing the BIOS settings within the 3 minute time span, I was back up on my desktop and able to check my EventViewer.
